## 🔒 施工约束（CLAUDE_CONSTRAINT_PLUGIN v1.0 自动生成）

### 八大铁律

1. **不越权** — 只做职责内的事
2. **不死循环** — 任何操作必须有停止条件
3. **思维透明化** — 展示多方案与权衡（规划/建设型任务强制）
4. **必须留痕** — 未写入 task_ledger = 违规执行
5. **必须可验收** — 无法验收 = 未完成
6. **代码极简化** — 最少必要代码，不堆抽象层
7. **修改精准化** — 仅改必要区域，不碰无关部分
8. **失败必须收敛** — 2 次失败必须进降级链路

### 八条执行纪律

| # | 纪律 |
|---|------|
| ① | task_ledger 为唯一权威源 |
| ② | 执行前必查 task_ledger 避免重复 |
| ③ | 已完成禁止重复执行 |
| ④ | 完成必写 output_summary + artifact_refs |
| ⑤ | 规划前必查历史同类任务 |
| ⑥ | 核心代码改动先报 MAIN |
| ⑦ | 基础设施/端口/配置改动→路由 MAIN |
| ⑧ | 收尾三步未完成不算完成 |

### BLD 专属红线

- ❌ 不碰 gateway、config.yaml、.env、密钥文件
- ❌ 不重启服务、不改 cron、不改系统配置
- ❌ 不改其他 agent 文件
- ❌ 不读或输出 API key/token/password/secret
- ❌ 超 2 次失败不无限重试
- ✅ 安全工具类任务：fake fixtures、仅 stdlib、只读扫描、输出不含密钥原文

### 失败处理

| 失败次数 | 动作 |
|----------|------|
| 1 次 | 记录事件，自动重试（最多 1 次） |
| 2 次 | 停止重试，上报 MAIN |
| MAIN 失败 | 上报 CY |
